In computer security, a vulnerability is a weakness in some software or system, which can be exploited by an attacker in order to perform unauthorized actions (such as remote code execution).

There some databases that collect those vulnerabilities are published. One known database is NVD - National Vulnerability Database.

Every entry in this database describes a vulnerability in human langue. The following assignment includes examples of vulnerabilities descriptions.

You should tag the words in the text according to the following labels:

1. Software or Service : Application/Software name, which contains a security vulnerability.
2. Version : The specific version of the vulnerable software.
3. Operating System : the operating system which runs the vulnerability software.
4. Network Protocol : the network protocol which the attacker utilized.
5. Port : specific port which the attacker utilized.
6. Means : Vulnerability type (for example - Buffer Overflow).
7. Impact : what can the attacker achieve by the attack (for example - run software. Achieve root permissions, etc.).
8. Hardware : the specific Hardware which includes the vulnerability.
9. Privilege : the required permission to exploit the vulnerability (for example - root/user).
10. Attack Vector : the context in which the vulnerability is exploitable (for example - physical access, remote access, sharing the same Lan, etc.).
11. Attack Technique :the specific action that the attacker performs (for example - send long HTTP request).
